Reconstruindo o backtest: de gatilhos para dummies + regressão

Um mapa visual do que vamos construir. O sistema legado mistura cálculo de indicadores, máquina de gatilhos e combinação de variáveis num só notebook. Aqui isolamos cada peça, trocamos os gatilhos por dummies de rompimento, e medimos cada modelo com regressão logística.

A ideia em uma frase

O que muda, e o que permanece igual ao legado.
Legado · v11.4.2
  • _?value → _?gatilho → persistência k dias → combinação de pares
  • Centenas de colunas, df pesado, difícil de auditar
  • Correlação de Pearson + médias de deslocamento
  • Tudo num notebook, sem testes
Rebuild

O que NÃO muda O mecanismo de deslocamento de datas (a "daylist") que cria a variável dependente é herdado do legado. É o coração que paraleliza os resultados — explicado logo abaixo.

O coração: deslocamento de datas → variável dependente

Cada dia vira uma amostra rotulada. Olhamos h dias à frente e perguntamos: o preço subiu? Sim → 1   Não → 0. Arraste o horizonte para ver o rótulo mudar.
Close mma (média móvel) rompimento (dia t = preditor) janela futura (h dias)
20 d Rótulo do dia do rompimento: 1 (subiu) 
# target.py — genérico, herdado do legado, NÃO depende do indicador
def add_labels(df, horizons):
    for h in horizons:                          # a "daylist"
        fwd = df["Close"].shift(-h) / df["Close"] - 1   # retorno futuro
        df[f"y_{h}d"] = (fwd > 0).astype("Int8")     # 1 subiu, 0 não
    return df
Cuidado científico O rótulo usa o futuro de propósito (é backtest). Mas as últimas h linhas ficam sem rótulo (NaN) e precisam ser descartadas — senão vazamos dados. O preditor (rompimento) só usa o passado/presente.

A variável independente: o rompimento

A dummy é 1 apenas no dia em que o Close cruza a mma de baixo para cima, respeitando uma tolerância. Nos demais dias, 0. É um evento — por isso raro.
# indicators/mma.py — o PLUG-IN. Troque este arquivo para mme, obv, ...
def build_signal(df, window=20, tol=0.0):
    mma   = df["Close"].rolling(window).mean()
    above = df["Close"] > mma * (1 + tol)        # estado: acima da banda
    cross = above & ~above.shift(1, fill_value=False) # evento: cruzou hoje
    return cross.astype("Int8")                  # a dummy 0/1

A arquitetura: indicador é um plug-in

Hoje mma. Amanhã você troca só o arquivo do indicador e roda mme ou obv. O resto não muda. O df permanece de um indicador só.
data.py
Carrega OHLCV de um ticker via yfinance (parametrizável por ticker/período).
target.py
Deslocamento de datas → rótulos y_{h}d 0/1. Genérico e fixo: independe do indicador.
indicators/mma.py
Produz a dummy de rompimento. É o que você troca para testar outro indicador. Mesma interface build_signal(df, **params).
modeling.py
Ajusta a logística (statsmodels) e devolve pseudo-R², coef, p-valor, n, n_eventos. Aceita 1 preditor agora, N depois (stepwise).
sweep.py
Percorre o grid e monta o summary — 2 linhas por modelo (logística + OLS). Não conhece o indicador.
run_mma.py
Entrypoint: liga tudo para o mma. Para mme, um run_mme.py quase idêntico.

Expectativa honesta Como você mesmo disse: um único indicador não tem valor preditivo real. Os pseudo-R² vão ser minúsculos — e tudo bem. O valor aqui é o arcabouço limpo e testável que serve de base para o futuro.
